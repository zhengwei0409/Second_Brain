# 2.4 — The Internet & Dot-com Era (1990s–2000s)

## 1. High-Level Map（这个主题在大局中的位置）

上一节（[[2.3 — The Personal Computing Era (1980s)]]）讲了一件大事：软件第一次变成了"可以独立打包卖钱的商品"（shrink-wrapped software）。但你有没有注意到，那个时代的软件有个巨大的局限——**每台 PC 都是一座孤岛**。

你买了一份 Word 处理软件装进你的电脑，我买了一份装进我的电脑，我们两台机器之间，**完全没办法互相说话**。软件再怎么厉害，也只能在"你自己的那台机器"里打转。

2.4 要讲的，就是这座座孤岛是怎么被一根网线连起来的——以及这根网线怎么把整个软件行业的规则，又一次彻底重写。这一节是从"2.3 消费软件"通往"2.5 移动云时代"的桥梁：没有互联网把电脑连起来，后面的"App Store"、"云计算"、"SaaS 订阅制"通通无从谈起。

---

## 2. Fundamentals First（从零开始，每个术语都讲透）

### 2.1 什么是 World Wide Web（万维网）？

先分清楚一个很多人搞混的概念：**互联网（Internet）**和**万维网（World Wide Web，简称 Web）不是一回事**。

- **Internet**：是"网线本身"——一整套让全世界的计算机能互相传输数据的基础设施和技术标准。这个东西其实在 1960-70 年代（还记得 Mainframe 时代吗）军方和大学之间就已经在研究了。
- **World Wide Web**：是**建立在 Internet 之上的一个"应用"**——1989 年由 Tim Berners-Lee 发明，用**网页（web page）**和**超链接（hyperlink）**的方式，把信息组织起来，让普通人可以用"点击"的方式在信息之间跳转。

**类比**：如果 Internet 是"全国的高速公路网"，那 Web 就是"跑在这些公路上的快递系统"——高速公路早就存在了，但直到"快递系统"（Web）出现，普通人才真正开始大量使用这套基础设施。

1990 年代，Web 走进了千家万户，"上网"这个动作，第一次不再是科学家和军方的专利，而是任何一个买了 PC、装了 Modem（调制解调器，把电脑信号转换成能在电话线上传输的信号）的普通人都能做的事。

### 2.2 Web Developer Emergence（网页开发者的出现）——一个全新职业诞生了

回想 2.3 讲的软件公司——他们写的软件是要**安装（install）**到你电脑里才能用的。但 Web 带来了一种全新的软件形态：**网页**。

写网页需要全新的技能：HTML（网页的结构）、后来加入的 CSS（网页的样式）、JavaScript（网页的互动逻辑）。这些技能和"写一个装在 Windows 上的软件"完全不是一回事。

于是，一个全新的职业出现了：**Web Developer（网页开发者）**。这批人专门给企业做"网站"——一开始可能只是一个"电子版的公司介绍手册"，后来逐渐演变成能买卖商品、能登录账号、能实时互动的复杂系统。

这是软件行业史上第一次，"你的产品要不要有个网站"变成了每家公司都要回答的问题——这也是"每家公司都是科技公司"这个说法（你会在 5.6 再次遇到）最早的起点。

### 2.3 Dot-com Boom & Bust（互联网泡沫的膨胀与破裂）

**".com"** 这个后缀你天天看到，但你知道为什么当年会有一大批公司抢着把名字后面加上 ".com" 吗？

1990 年代中后期，投资人突然意识到：**互联网能连接全世界的人，谁能第一个占领这个新市场，谁就能发大财**。于是资本疯狂涌入任何和互联网沾边的公司——哪怕这家公司根本还没有一个能赚钱的商业模式，只要名字里带 ".com"，股价就能被炒上天。这就是 **Dot-com Boom（互联网泡沫的"膨胀"阶段）**，大约在 1995-2000 年。

问题是：**很多这些公司根本不知道怎么把"用户"变成"收入"**。他们烧了大把投资人的钱去打广告、抢用户，却没有想清楚商业模式。到了 2000 年，投资人开始清醒过来，一夜之间集体撤资，成千上万家".com"公司瞬间倒闭、股价归零——这就是 **Dot-com Bust（泡沫破裂）**。

**为什么这对你很重要？** 因为这是软件史上第一次大规模的"资本 vs 技术"教训：技术再新、故事再性感，如果没有真实的商业模式支撑，泡沫迟早会破。这个教训直到今天还在被反复引用——包括后面 2.6 你会学到的 AI 时代，也有人在问"这次会不会是另一场 Dot-com Bust？"

### 2.4 Client-Server Model（客户端-服务器模型）

这是一个技术架构概念，解释了"网页到底是怎么运作的"。

- **Client（客户端）**：就是你的电脑、你的浏览器——**发出请求**的那一方（"给我看看这个网页！"）
- **Server（服务器）**：是某家公司放在机房里的计算机——**接收请求、处理、然后把结果传回来**的那一方（"好，这是你要的网页内容"）

**类比**：这就像你在餐厅点餐——你（Client）跟服务员说"我要一份牛排"，厨房（Server）把牛排做好，端回来给你。你不需要知道厨房里具体怎么运作，你只需要"发请求、等结果"。

这个模型之所以重要，是因为它和 2.3 讲的"shrink-wrapped software"是**完全不同的软件运作方式**：
- **Shrink-wrapped software**：软件全部装在你自己电脑里跑，不需要联网
- **Client-Server**：软件的"大脑"（数据、逻辑）放在远方某台服务器上，你的电脑只是负责"显示结果"和"发送请求"

这个架构思想，直接是后面 2.5（云计算）的地基——你现在用手机 App 时，数据存在"云端"，本质上就是 Client-Server 模型的延伸。

### 2.5 Open Source Movement（开源运动）

最后一个概念，讲一个和"卖软件赚钱"完全相反方向的运动。

2.3 讲过，PC 时代软件是靠"打包卖钱"（shrink-wrapped software）赚钱的——你想用这个软件，就必须付钱买。但 1990 年代，随着互联网让"分享代码"变得前所未有的容易，一群程序员开始提倡另一种理念：**Open Source（开源）**——把软件的**源代码（source code，也就是程序员写的、人类能读懂的原始代码）公开**，任何人都可以免费查看、使用、修改、甚至重新发布。

代表性的例子是 **Linux**（一个开源的操作系统）——它不属于任何一家公司，而是由全世界成千上万名志愿程序员一起协作写出来的。

这在当时是个很反直觉的商业模式：**"免费公开代码，你要怎么赚钱？"** 但开源运动证明了：即使代码免费，围绕这些代码依然可以发展出咨询服务、企业支持、云端托管等各种商业模式。这也是为什么今天几乎所有软件工程师的日常工作都离不开开源工具（比如你正在用的 Git、VS Code 的很多底层技术）——开源已经成为整个软件产业的"地基"，而不是一个小众运动。

## 3. Progressive Explanation（把这些概念串起来）

现在把这几块拼图拼起来，讲一个完整的故事：

> 1990 年代，建立在 **Internet** 基础设施之上的 **World Wide Web** 走进了千家万户，让普通人第一次能用"点击链接"的方式浏览全世界的信息。这套新技术运作的方式，靠的是 **Client-Server Model**——你的电脑（client）发请求，某台远方的服务器（server）处理并回传结果。
>
> 因为网页需要全新的技能（HTML、JavaScript），一个全新的职业——**Web Developer**——应运而生，专门为企业打造网站。
>
> 资本市场很快嗅到了这股新浪潮的味道，疯狂把钱砸向任何和互联网沾边的公司，催生了 **Dot-com Boom**。但很多公司只有"故事"没有真实的商业模式，最终在 2000 年前后集体崩盘，酿成 **Dot-com Bust**——软件史上第一次"资本狂热撞上现实"的惨痛教训。
>
> 与此同时，在商业世界疯狂追逐利润的另一侧，**Open Source Movement** 走出了一条完全相反的路：把代码免费公开，靠社区协作和衍生服务来创造价值。这两股力量——**疯狂逐利的 Dot-com 公司**和**理想主义的开源社区**——共同塑造了今天软件行业的样貌：既有靠订阅和广告赚取巨额利润的科技巨头，也有支撑起整个行业地基的免费开源工具。

## 4. 5 个你必须完全掌握的核心概念

### ① World Wide Web（万维网）
- **定义**：建立在 Internet 基础设施之上、用网页和超链接组织信息的应用系统，让普通人可以通过点击在信息间跳转。
- **为什么重要**：它是"上网"这个行为第一次走进普通家庭的关键技术，也是后面所有互联网软件（网站、Web App、云服务）的起点。
- **自测题**：Internet 和 World Wide Web 有什么区别？
  **答案**：Internet 是让全球计算机互相传输数据的底层基础设施；World Wide Web 是建立在 Internet 之上、用网页和超链接组织信息的一个应用，两者不是同一个东西。

### ② Web Developer Emergence（网页开发者的出现）
- **定义**：1990 年代因为网页技术（HTML/CSS/JavaScript）兴起而诞生的全新职业，专门负责设计和搭建网站。
- **为什么重要**：这是第一次"公司要不要有个网站"变成企业的必答题，也标志着软件开发从"只给单机装软件"扩展出了"给网络做网页"这个全新分支。
- **自测题**：Web Developer 和 2.3 时代写 shrink-wrapped software 的程序员，用的技能一样吗？
  **答案**：不一样。Shrink-wrapped software 的程序员写的是安装到用户电脑里独立运行的程序；Web Developer 用 HTML、CSS、JavaScript 等技术专门开发跑在浏览器里、通过网络访问的网页。

### ③ Dot-com Boom & Bust（互联网泡沫的膨胀与破裂）
- **定义**：1990 年代末投资人疯狂追捧任何和互联网相关的公司（Boom），但因为许多公司缺乏真实商业模式，在 2000 年前后集体倒闭崩盘（Bust）。
- **为什么重要**：它是软件史上第一次"资本狂热 vs 现实商业模式"的惨痛教训，提醒我们技术热潮不等于可持续的生意，这个教训至今仍被反复用来对照后续的科技热潮（比如 AI）。
- **自测题**：Dot-com Bust 崩盘的根本原因是什么？
  **答案**：许多".com"公司虽然吸引了大量投资和用户，却没有想清楚如何把用户转化为真实收入，缺乏可持续的商业模式，一旦投资人撤资信心崩溃，公司就迅速倒闭。

### ④ Client-Server Model（客户端-服务器模型）
- **定义**：一种软件架构，客户端（如你的浏览器）发送请求，服务器（远方的计算机）处理请求并返回结果。
- **为什么重要**：它是网页和几乎所有互联网软件运作的底层架构原理，也是后面云计算（2.5）"数据存在云端"这个概念的直接源头。
- **自测题**：在 Client-Server Model 中，浏览器和网站背后的服务器分别扮演什么角色？
  **答案**：浏览器（client）负责发送请求（比如"给我这个网页"）并显示结果；服务器（server）负责接收请求、处理数据和逻辑，然后把结果传回给浏览器。

### ⑤ Open Source Movement（开源运动）
- **定义**：一场提倡把软件源代码公开、允许任何人免费查看、使用、修改和分发的运动，代表作品如 Linux。
- **为什么重要**：它证明了即使代码免费公开，依然可以围绕它发展出可持续的商业模式（如企业支持、云端托管），并让开源工具成为了今天整个软件行业不可或缺的地基。
- **自测题**：开源软件免费公开代码，那开源相关的公司或社区是怎么维持运作、创造价值的？
  **答案**：虽然代码本身免费，但可以通过提供企业级技术支持、咨询服务、云端托管、认证培训等衍生服务来创造收入，同时依靠全球志愿者社区协作来持续改进软件本身。
