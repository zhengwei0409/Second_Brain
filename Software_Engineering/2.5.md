# 2.5 — The Mobile & Cloud Era (2007–2019)

## 1. High-Level Map（这个主题在大局中的位置）

上一节（[[2.4 — The Internet & Dot-com Era (1990s–2000s)]]）讲到，Web 让电脑之间第一次能"说话"了——但注意，那时候的上网方式仍然是"坐在书房里，打开一台台式机"。互联网虽然连起来了，但**人还是被绑在一个固定的地方**才能上网。

2.5 要讲的，是两场几乎同时发生、又互相纠缠在一起的革命：

1. **手机革命（Mobile）**：互联网从"书房里的台式机"搬进了"你口袋里的手机"——软件第一次变成了随时随地、24 小时跟着你走的东西。
2. **云计算革命（Cloud）**：软件公司不再需要自己买机房、买服务器——直接向亚马逊"租"一整套计算能力。

这两场革命看似各自独立，其实是**一体两面**：手机把"用户"从书桌前解放出来，云计算把"公司"从机房里解放出来。两边一起解放，才催生出你现在天天在用的东西——Instagram、Uber、Netflix、微信——这些 App 背后如果没有云计算撑腰，手机再怎么普及也跑不动。

这一节是承上启下的关键桥梁：没有 2.5 打下的地基（App、云、SaaS），后面 2.6 的 AI 时代根本无从谈起——今天你用的 AI 产品，几乎清一色跑在云端服务器上，通过手机 App 或网页触达你。

## 2. Fundamentals First（从零开始，每个术语都讲透）

### 2.1 iPhone / App Store（2007–2008）——手机革命的起点

先问一个问题：**在 2007 年之前，手机不能上网吗？** 其实可以，但体验极差——屏幕小、按键式操作、网速慢，那时候的手机（比如 Nokia）本质上还是"能打电话、能发短信"的工具，上网只是个勉强能用的附加功能。

**2007 年 1 月，Steve Jobs 在台上发布了 iPhone。** 它做对了几件事，彻底改写了"手机"这个词的定义：

- **触屏（touchscreen）**：不再需要实体键盘，手指直接在屏幕上滑动、点击。
- **完整的浏览器**：iPhone 上的 Safari 浏览器，第一次能显示"和电脑上一模一样"的完整网页，而不是阉割版。
- **把电脑装进口袋**：iPhone 本质上是一台迷你电脑，只是恰好也能打电话。

但真正引爆这场革命的，其实是 2008 年发布的 **App Store**。在这之前，手机上能用的软件非常有限，而且很难获取。App Store 做的事情很简单但极具颠覆性：**任何开发者都可以写一个 App，上传到 App Store，全世界的 iPhone 用户都能直接下载安装**。

**类比**：如果说 2.3（PC 时代）的"shrink-wrapped software"是"你要开车去 Best Buy 买一盒软件光碟回家装"，那 App Store 就是"你在口袋里的商店，动动手指，30 秒后软件就装好了"。这个"分发（distribution）"的门槛，被降到了历史最低点。

紧接着 2008 年，Google 也推出了自己的手机操作系统 **Android**，并配上了自己的应用商店（Google Play）。从此手机市场变成了 iOS 和 Android 两强争霸的局面——这个格局你会在 [[3.6 — Mobile Engineering]] 里再深入认识。

### 2.2 Mobile-First（移动优先）——一个新的设计哲学

App Store 火了之后，工程师和设计师们很快发现一个问题：**过去十几年，大家都是在为"大屏幕、有键盘鼠标"的电脑设计软件**。但手机屏幕小、只能用手指戳、而且用户经常是在走路、在地铁上、网络信号不稳定的情况下使用——这是完全不同的使用场景。

于是诞生了 **Mobile-First（移动优先）** 这个设计与开发理念：**先为手机这个"最受限制"的设备设计产品，再考虑怎么扩展到电脑等大屏幕**。逻辑是——如果你的产品在小屏幕、慢网速、手指操作的最苛刻条件下都能用得顺畅，那放大到电脑屏幕只会更简单；反过来，如果先为电脑设计再硬塞进手机，往往体验很糟糕。

这不只是一个技术选择，它彻底改变了公司招人和组建团队的方式——从这个时代开始，"我们要不要有专门的 Mobile 团队"变成了几乎每家科技公司都要回答的问题。

### 2.3 Cloud Computing（云计算）——公司不用再自己买机房了

现在切换到这场革命的另一半：**云计算**。

先回想一下：在这之前，如果你想开一家网络公司（比如做一个网站），你需要做什么？你得自己买很多台服务器（server，就是专门用来跑软件、存数据的电脑），把它们放进一个有空调、有备用电源、有网络专线的机房里，还要雇人 24 小时看着这些机器，随时修理故障——这一整套东西极其昂贵，而且是**在你的网站还没有一个用户之前**，就要先砸这么多钱。

**2006 年，亚马逊（Amazon）推出了 AWS（Amazon Web Services）。** 核心的想法是：亚马逊自己为了运营网购业务，早就建好了全世界数一数二庞大的机房和服务器群——那**为什么不把这些多余的计算能力，像水电一样按用量出租给其他公司呢？**

这就是 **Cloud Computing（云计算）**：你不需要自己买服务器，而是像交水电费一样，按你实际用掉的计算资源，向亚马逊（或后来的 Google Cloud、微软 Azure）付钱。你今天流量小，就少付一点；明天你的 App 一夜爆红，流量暴增十倍，你只需要在网页上点几下"扩容"，几分钟内云端就能自动帮你多开几台服务器——这在过去"自己买机房"的年代是不可能做到的。

**类比**：这就像"自己买发电机 vs 直接用国家电网"。以前每家公司都得自备发电机（自己买机房），现在只要插上电网插座（连上云计算），用多少付多少，随时想加大用电量都行。

### 2.4 IaaS（Infrastructure as a Service，基础设施即服务）

这是云计算里最基础的一层服务，值得单独讲清楚。

**IaaS** 指的是：云服务商只帮你把最底层的"硬件"变成可以租用的东西——比如虚拟的服务器（用来跑你的程序）、虚拟的硬盘（用来存数据）、网络连接。至于要在这些硬件上装什么操作系统、跑什么软件、怎么写你的程序逻辑，**都还是你自己的事**。

**类比**：IaaS 就像租一间"毛坯房"——房东（云服务商）给你四面墙、水电接口，但室内装修、家具摆设，全部由你自己搞定。这跟"精装修拎包入住"（我们后面会讲到的 SaaS）是完全不同的服务层次。

AWS 一开始最核心的产品，就是叫做 **EC2** 的 IaaS 服务——本质上就是"云端的虚拟电脑，你想要多少台、多大配置，自己按需租"。

### 2.5 SaaS（Software as a Service，软件即服务）

云计算除了让公司能"租用底层硬件"（IaaS），还催生了一种全新的**卖软件的方式**——SaaS。

回想 2.3 讲的 shrink-wrapped software：你花钱**买断**一份软件（比如买一套 Microsoft Office 光碟），装进你的电脑，这份软件就永远是你的了（除非你换电脑要重装）。

**SaaS 完全不是这个逻辑**。SaaS 公司把软件放在**云端服务器**上运行，你不需要"安装"任何东西，只需要打开浏览器或者一个很轻的 App，然后**按月或按年付订阅费**，就能持续使用这个软件。你今天用的 Netflix、Spotify、Notion、微信读书会员，本质上都是 SaaS。

**为什么这个转变对软件公司来说是革命性的？** 因为它把公司的收入模式，从"一次性卖断"变成了"持续订阅"——这意味着公司必须不断更新、维护、优化产品，才能留住每个月还在付钱的用户，而不是像以前那样"卖完一套光碟就完事了"。这也直接导致了后面你会在 [[7.1 — Everyday Work Vocabulary]] 遇到的很多工程文化——比如"持续发布新版本"变成了行业常态，而不是"一年才出一个大版本"。

## 3. Progressive Explanation（把这些概念串起来）

现在把这几块拼图拼起来，讲一个完整的故事：

> 2007 年，**iPhone** 的发布，加上 2008 年 **App Store** 的推出，第一次让"软件"变成了可以随时装进口袋、随时下载安装的东西。这场手机革命，催生了 **Mobile-First** 这个全新的设计哲学——工程师和设计师必须先为小屏幕、手指操作的苛刻条件设计产品，而不是把电脑软件硬塞进手机。
>
> 与此同时，另一场同样重要的革命正在服务器背后悄悄发生：2006 年 **AWS** 的推出，让 **Cloud Computing** 变成现实——公司不再需要自己砸钱买机房，而是像交水电费一样，按需向亚马逊租用计算资源。这套"云端硬件出租"的模式被称为 **IaaS**。
>
> 云计算不仅改变了"公司怎么运营基础设施"，也彻底改变了"软件怎么卖"——过去一次性买断的模式，逐渐被 **SaaS**（订阅制、跑在云端、免安装）取代。
>
> 这两场革命合在一起产生了化学反应：因为有了云计算撑腰，一家创业公司不再需要先砸几百万买机房才能上线产品，而是可以立刻租用 AWS 的服务器，把产品直接做成 App 放上 App Store，让全世界的手机用户下载。**这就是为什么 2007-2019 年间，我们看到了 Uber、Instagram、Airbnb 这样"几个人的团队 + 一部手机 App + 云端服务器"就能颠覆整个传统行业的奇迹**——在 PC 时代，这种事情的资本门槛是不可能做到这么低的。

## 4. 5 个你必须完全掌握的核心概念

### ① iPhone / App Store（2007–2008）
- **定义**：2007 年发布的 iPhone 用触屏和完整浏览器重新定义了手机，2008 年推出的 App Store 让任何开发者都能把软件直接分发给全世界的手机用户。
- **为什么重要**：它把"分发软件"的门槛降到历史最低，是整个移动互联网产业爆发的起点，也直接催生了 [[3.6 — Mobile Engineering]] 这个全新工程领域。
- **自测题**：App Store 相比 2.3 时代的 shrink-wrapped software，在"软件分发"这件事上做对了什么？
  **答案**：过去买软件需要去实体店购买光碟、自己安装，App Store 让开发者可以直接把软件上传到一个全世界用户都能访问的商店，用户动动手指几十秒就能下载安装，把分发门槛降到了历史最低。

### ② Mobile-First（移动优先）
- **定义**：一种设计与开发理念——优先为手机这种小屏幕、手指操作、网络不稳定的最受限设备设计产品，再考虑扩展到电脑等大屏幕。
- **为什么重要**：它改变了整个行业设计和开发产品的默认思路，也是"是否需要专门的 Mobile 团队"这个组织决策的源头。
- **自测题**：为什么要"先为手机设计，再扩展到电脑"，而不是反过来？
  **答案**：因为手机的屏幕小、只能用手指操作、网络环境也更不稳定，是最苛刻的使用场景；如果产品在这种最受限条件下都能用得顺畅，扩展到功能更强的电脑屏幕会容易得多；反过来先为电脑设计再硬塞进手机，往往体验很差。

### ③ Cloud Computing / AWS（云计算）
- **定义**：由亚马逊在 2006 年通过 AWS 率先推广的模式——公司不用自己买服务器建机房，而是按实际使用量向云服务商租用计算资源。
- **为什么重要**：它大幅降低了创业公司启动一个网络服务所需的前期资本，是 2007-2019 年间无数"轻资产"科技公司能够快速崛起的基础设施基石。
- **自测题**：云计算出现之前，一家公司想上线一个网站，必须先做什么？云计算之后呢？
  **答案**：之前必须自己买服务器、租机房、雇人 24 小时维护，属于巨大的前期投入；云计算之后，只需要按实际用量向 AWS 等云服务商付费租用，可以随时按需扩容或缩减，不需要任何前期硬件投入。

### ④ IaaS（Infrastructure as a Service）
- **定义**：云计算中最基础的一层服务，云服务商只提供虚拟服务器、存储、网络等底层硬件资源，操作系统之上的所有软件和逻辑仍由使用者自己搭建。
- **为什么重要**：它是 AWS 最早、最核心的商业模式（如 EC2），理解 IaaS 是理解云计算分层体系（IaaS / PaaS / SaaS）的第一步。
- **自测题**：用"租房"来类比，IaaS 提供的是什么样的房子？
  **答案**：IaaS 就像租一间"毛坯房"——云服务商只给你四面墙和水电接口（虚拟服务器、存储、网络），至于要装修成什么样、摆什么家具（安装什么系统、跑什么程序），都要使用者自己动手。

### ⑤ SaaS（Software as a Service）
- **定义**：一种软件销售模式——软件运行在云端服务器上，用户不需要安装，只需按月或按年订阅付费即可持续使用。
- **为什么重要**：它把软件行业的收入模式从"一次性卖断"转变为"持续订阅"，直接改变了软件公司必须不断更新维护产品的工作文化，是今天 Netflix、Notion 等主流产品的商业模式基础。
- **自测题**：SaaS 和 2.3 时代"买断式"的 shrink-wrapped software，最根本的区别是什么？
  **答案**：Shrink-wrapped software 是一次性付费买断、软件装在你自己电脑里离线运行；SaaS 是持续订阅付费、软件运行在云端服务器上，用户通过网络访问，停止订阅就无法继续使用。
