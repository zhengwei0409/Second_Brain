# 1.4 — Arrange-Act-Assert

## 在大局中的位置

上一节（1.3）你学了一个 test case 的所有组成部分：precondition、input、expected output、assertion 等。

这一节要解决的问题是：**怎么把这些部分组织成一个结构清晰、好读懂的测试？**

答案就是 **AAA Pattern（Arrange-Act-Assert）** — 把测试强制分成三个区块。

```
软件测试
├── [1.1] 为什么软件会失败？
├── [1.2] 什么是软件测试？
├── [1.3] 一个测试用例由什么构成？
└── [1.4] 怎么把测试写得结构清晰？← 你在这里
```

## 1. 为什么需要 AAA Pattern？

先看一个没有结构的测试：

```python
def test_checkout():
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    cart.add_item("banana", price=1.5, qty=3)
    user = User(balance=20.0)
    total = cart.get_total()
    user.deduct(total)
    assert user.balance == 11.5
    assert cart.is_empty() == False
```

这个测试能跑，但你要花时间才能搞清楚：哪些是准备工作？哪里是被测的动作？哪里是在验证结果？

**AAA Pattern 的目的：让测试一眼就能看懂。**

---

## 2. Arrange — 准备阶段

**Arrange** 是测试的第一个区块，负责 **搭建测试所需的一切初始状态**。

它对应 1.3 里的 **Precondition + Input 的准备工作**。

Arrange 里做的事：
- 创建被测的对象
- 准备输入数据
- 设置初始状态

```python
# Arrange
cart = ShoppingCart()
cart.add_item("apple", price=3.0, qty=2)
cart.add_item("banana", price=1.5, qty=3)
```

**Arrange 阶段不调用被测功能本身。** 你只是在搭舞台，还没开始表演。

---

## 3. Act — 执行阶段

**Act** 是测试的第二个区块，负责 **调用你正在测试的那个功能**。

这一行通常只有 **一行代码**。

```python
# Act
total = cart.get_total()
```

Act 阶段的关键：
- **只做一件事** — 触发被测行为
- 把结果存起来，留给 Assert 阶段用

如果 Act 阶段有多行，说明你可能在一个测试里测了多个行为，这违反了 Single Behavior Focus（后面讲）。

---

## 4. Assert — 验证阶段

**Assert** 是测试的第三个区块，负责 **检查 Act 产生的结果是否符合预期**。

它对应 1.3 里的 **assertion**。

```python
# Assert
assert total == 10.5
```

Assert 阶段的关键：
- 只验证 Act 那一步的结果
- **通常只有一个 assertion**（原因见 Single Behavior Focus）
- 如果 assertion 失败，测试框架会报错并告诉你

---

## 5. 完整例子：从混乱到清晰

**没有 AAA 的版本（难读）：**

```python
def test_get_total():
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    cart.add_item("banana", price=1.5, qty=3)
    total = cart.get_total()
    assert total == 10.5
```

**有 AAA 的版本（清晰）：**

```python
def test_get_total():
    # Arrange
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    cart.add_item("banana", price=1.5, qty=3)

    # Act
    total = cart.get_total()

    # Assert
    assert total == 10.5
```

代码完全一样，但加了三个区块注释之后，任何人读到这个测试，立刻就知道：
- "哦，前面是准备工作"
- "这一行是被测的功能"
- "最后这行是在验证结果"

**AAA 的价值不是让测试跑得更快，而是让测试更容易被人读懂和维护。**

---

## 6. Single Behavior Focus（单一行为原则）

AAA Pattern 有一个配套原则：**一个测试只测一个行为**。

**为什么？**

假设你在一个测试里放了两个 assertion：

```python
def test_checkout():
    # Arrange
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    user = User(balance=20.0)

    # Act
    user.checkout(cart)

    # Assert
    assert user.balance == 14.0   # assertion 1
    assert cart.item_count() == 0  # assertion 2
```

如果测试失败，你知道是哪个 assertion 出问题了吗？可能是 balance 算错了，也可能是购物车没清空，也可能两个都错。你需要花时间去查。

**每个测试只专注一个行为的好处：**

| 情况 | 单一行为 | 多个行为 |
|------|---------|---------|
| 测试失败时 | 立刻知道哪里出问题 | 需要调查是哪个地方错了 |
| 测试名字 | 清晰，一句话描述 | 很难起一个准确的名字 |
| 维护 | 改一个功能只影响相关测试 | 改一个功能可能影响很多测试 |

**实际操作：** 与其写一个测试同时验证 balance 和 cart，不如拆成两个测试：

```python
def test_checkout_deducts_balance():
    # Arrange
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    user = User(balance=20.0)
    # Act
    user.checkout(cart)
    # Assert
    assert user.balance == 14.0

def test_checkout_clears_cart():
    # Arrange
    cart = ShoppingCart()
    cart.add_item("apple", price=3.0, qty=2)
    user = User(balance=20.0)
    # Act
    user.checkout(cart)
    # Assert
    assert cart.item_count() == 0
```

现在每个测试失败时，你立刻知道是哪个行为出了问题。

## Self-Check

**Q1.** 以下代码，哪些行属于 Arrange？哪行属于 Act？哪行属于 Assert？

```python
def test_apply_coupon():
    cart = ShoppingCart()
    cart.add_item("shirt", price=50.0)
    coupon = Coupon(discount=0.1)

    discounted = cart.apply_coupon(coupon)

    assert discounted == 45.0
```

**A1.**
- **Arrange**（第 2-4 行）：`cart = ShoppingCart()`、`cart.add_item(...)`、`coupon = Coupon(...)`
- **Act**（第 6 行）：`discounted = cart.apply_coupon(coupon)`
- **Assert**（第 8 行）：`assert discounted == 45.0`

---

**Q2.** 为什么 Act 阶段通常只有一行代码？

**A2.** 因为 Act 阶段只做一件事：**触发被测的那一个行为**。如果 Act 有多行，说明你在一个测试里触发了多个行为，违反了 Single Behavior Focus 原则，测试失败时你就无法立刻定位是哪个行为出了问题。

---

**Q3.** 下面这个测试有什么结构上的问题？

```python
def test_user_registration():
    user = register_user("alice", "password123")
    assert user.id is not None
    assert user.username == "alice"
    assert user.is_active == True
    assert send_welcome_email_called == True
```

**A3.** 这个测试违反了 **Single Behavior Focus** 原则。它同时验证了 4 件事：user 有 id、username 正确、账号是激活状态、欢迎邮件被发送。如果测试失败，你不知道是哪一个出了问题。应该拆成 4 个独立的测试，每个只有一个 assertion。

---

**Q4.** AAA Pattern 和 1.3 里学的 test case 解剖有什么关系？

**A4.** 它们描述的是同一件事，只是视角不同：

| Test Case Anatomy (1.3) | AAA Pattern (1.4) |
|------------------------|-------------------|
| Precondition + Input 准备 | **Arrange** |
| 运行被测功能 | **Act** |
| Expected Output + Assertion | **Assert** |

AAA Pattern 是 test case 结构在代码层面的具体实现方式。
